## 基礎知識

> **例で使うテーブルについて:** 以下の例では架空の `books`（書籍）テーブルを使います。演習問題で使う `employees` テーブルとは異なりますが、SQL の書き方は同じです。

### 1. SELECT 文の基本構文

```sql
SELECT カラム1, カラム2 FROM テーブル名;
```

SQL の基本は「**どのテーブルから（FROM）**」「**何を取得するか（SELECT）**」の2つを指定することです。

```sql
-- books テーブルから title と price を取得する
SELECT title, price FROM books;
```

| title          | price |
|----------------|-------|
| SQL入門        | 2800  |
| Python基礎     | 3200  |
| データ分析実践 | 3800  |
| …              | …     |

### 2. 全カラムの取得（`*`）

`*`（アスタリスク）を使うと、すべてのカラムを取得できます。

```sql
SELECT * FROM books;
```

| id | title          | genre_id | price |
|----|----------------|----------|-------|
|  1 | SQL入門        | 1        | 2800  |
|  2 | Python基礎     | 1        | 3200  |
| …  | …              | …        | …     |

探索や確認には便利ですが、本番コードでは必要なカラムを明示するのが一般的です。

### 3. カラムエイリアス（`AS`）

`AS` を使うとカラムに別名（エイリアス）をつけられます。
日本語のカラム名を使いたいときや、計算結果に名前をつけるときに便利です。

```sql
SELECT title AS タイトル, price AS 価格 FROM books;
```

| タイトル       | 価格  |
|----------------|-------|
| SQL入門        | 2800  |
| Python基礎     | 3200  |
| データ分析実践 | 3800  |
| …              | …     |

### 4. 文字列結合（`||`）

PostgreSQL では `||` 演算子で文字列を結合できます。

```sql
SELECT title || '（在庫あり）' AS 表示名 FROM books;
```

| 表示名                    |
|---------------------------|
| SQL入門（在庫あり）       |
| Python基礎（在庫あり）    |
| データ分析実践（在庫あり）|
| …                         |

### 5. 計算式

SELECT の中で四則演算を使えます。

```sql
SELECT title, price * 1.1 AS 税込価格 FROM books;
```

| title          | 税込価格 |
|----------------|----------|
| SQL入門        | 3080.0   |
| Python基礎     | 3520.0   |
| データ分析実践 | 4180.0   |
| …              | …        |

### 6. DISTINCT による重複除去

`DISTINCT` を使うと重複した値を取り除いた結果が得られます。

```sql
SELECT DISTINCT genre_id FROM books;
```

| genre_id |
|----------|
| 1        |
| 2        |
| 3        |
| NULL     |

### 問題 1-6: DISTINCT による重複除去

**ファイル:** `exercises/chapter01/ex06.sql`

`employees` テーブルに存在する **部署ID（`dept_id`）の一覧** を重複なく取得してください。

**期待される結果（5行）:**

| dept_id |
|---------|
| 1       |
| 2       |
| 3       |
| 4       |
| NULL    |

> ※ `NULL` は「どの部署にも属していない」社員がいることを示します。
> ※ `ORDER BY dept_id` を加えると上記の順序で並びます（PostgreSQL では NULL は昇順の末尾に来ます）。
