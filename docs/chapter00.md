# Chapter 00: はじめに — データベースと SQL の基礎知識

このチャプターは演習なしの読み物です。
Chapter 01 から手を動かす前に、必要な概念を整理します。

---

## 1. データベースとは

**データベース**とは、大量のデータを整理して保存・検索・更新できる仕組みです。

身近な例として、銀行の口座情報、ECサイトの商品一覧、スマホのアドレス帳なども、すべてデータベースで管理されています。ただし「ファイルに保存するだけ」と何が違うのかというと、データベースは次のことが得意です。

- **高速な検索**: 何百万件のデータからでも条件に合う行を素早く取り出せる
- **整合性の保証**: 矛盾したデータが入らないようルールを設けられる
- **同時アクセス**: 複数のユーザーが同時に読み書きしても壊れない

---

## 2. リレーショナルデータベース

データベースにはいくつか種類がありますが、業務システムで最も広く使われているのが**リレーショナルデータベース（RDB）**です。

RDB の特徴は、データを**表（テーブル）**の形式で管理することです。また、複数のテーブルを「関係（リレーション）」でつなぎ合わせることで、複雑なデータを無駄なく表現できます。

代表的な RDB のソフトウェア（RDBMS）には PostgreSQL、MySQL、SQLite、Oracle などがあります。このコースでは **PostgreSQL** を使います。

---

## 3. テーブル・行・カラム

RDB でデータを格納する単位が**テーブル**です。テーブルは Excel のシートに似た構造を持っています。

```
┌────┬──────┬─────────┬────────┐
│ id │ name │ dept_id │ salary │   ← カラム（列）
├────┼──────┼─────────┼────────┤
│  1 │ 花子 │       1 │  75000 │   ┐
│  2 │ 一郎 │       1 │  60000 │   ├ 行（レコード）
│  3 │ 美子 │       2 │  55000 │   ┘
└────┴──────┴─────────┴────────┘
```

| 用語 | 別名 | 説明 |
|------|------|------|
| テーブル | 表 | データをまとめたひとかたまり |
| カラム | 列・フィールド | データの種類（name、salary など） |
| 行 | レコード・タプル | 1件分のデータ |

---

## 4. NULL とは

**NULL** は「値が存在しない・不明である」ことを表す特別な値です。

`0`（ゼロ）や `''`（空文字列）とは異なります。たとえば「給与が 0 円」と「給与がまだ決まっていない」は別の状態であり、後者を NULL で表します。

このコースのデータでは、たとえば「まだ部署が決まっていない社員」の `dept_id` が NULL になっています。

---

## 5. 主キーと外部キー

### 主キー（PRIMARY KEY）

テーブル内の**行を一意に識別**するカラムです。このコースのテーブルはすべて `id` カラムが主キーで、同じ値が重複することはありません。

### 外部キー（FOREIGN KEY）

**別テーブルの主キーを参照**するカラムです。たとえば `employees.dept_id` は `departments.id` を参照しており、「この社員はどの部署に所属しているか」という関係を表します。

```
departments              employees
──────────               ─────────────────────
id  ←──────────────── dept_id
name                     name
                         salary
```

外部キーを使ってテーブルどうしを結びつけることで、同じ情報を複数の場所に重複して持たずに済みます（たとえば部署名は `departments` テーブルに1か所だけ保存し、社員テーブルは部署の `id` だけを持つ）。

## 6. SQL とは

**SQL**（Structured Query Language）は、データベースを操作するための言語です。主に次の4種類の操作を行います。

| 操作 | SQL 文 | 説明 |
|------|--------|------|
| 取得 | `SELECT` | データを読み出す |
| 追加 | `INSERT` | 新しい行を追加する |
| 更新 | `UPDATE` | 既存の行を書き換える |
| 削除 | `DELETE` | 行を削除する |

Chapter 01〜06 は主に `SELECT`（読み出し）を扱います。Chapter 07 で `INSERT` / `UPDATE` / `DELETE` を学びます。

SQL は大文字・小文字を区別しませんが（`SELECT` と `select` は同じ意味）、このコースでは予約語を大文字で書く慣習に統一します。

## 7. このコースで使うテーブル

架空の会社の社内システムを題材に、4つのテーブルを使います。

### employees（社員）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | 整数 | 社員ID（主キー） |
| name | 文字列 | 氏名 |
| dept_id | 整数 | 所属部署ID（NULL = 未配属） |
| salary | 整数 | 給与（円） |
| hire_date | 日付 | 入社日 |
| manager_id | 整数 | 直属の上司の社員ID（NULL = 上司なし） |

| id | name   | dept_id | salary | hire_date  | manager_id |
|----|--------|---------|--------|------------|------------|
| 1  | 花子   | 1       | 75000  | 2020-04-01 | NULL       |
| 2  | 一郎   | 1       | 60000  | 2021-06-15 | 1          |
| 3  | 美子   | 2       | 55000  | 2019-03-01 | NULL       |
| 4  | 悠介   | 2       | 50000  | 2022-01-10 | 3          |
| 5  | 由子   | 3       | 65000  | 2018-09-01 | NULL       |
| 6  | 健太   | 1       | 80000  | 2017-05-20 | 1          |
| 7  | あかね | 4       | 48000  | 2023-02-14 | NULL       |
| 8  | 昭二   | NULL    | 45000  | 2023-07-01 | NULL       |
| 9  | 京子   | 3       | 58000  | 2021-11-30 | 5          |
| 10 | 翔太   | 4       | 52000  | 2020-08-15 | 7          |

> `dept_id` の `1` が「開発」、`2` が「マーケティング」…と対応しています。どの部署かは `departments` テーブルで確認できます（Chapter 05 以降で使います）。
> `manager_id` の `1` は「上司が社員ID=1（花子）」という意味です。NULL は「上司がいない」（組織のトップなど）を表します。

### departments（部署）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | 整数 | 部署ID（主キー） |
| name | 文字列 | 部署名 |
| location | 文字列 | 所在地 |

| id | name           | location |
|----|----------------|----------|
| 1  | 開発           | 東京     |
| 2  | マーケティング  | 大阪     |
| 3  | 人事           | 東京     |
| 4  | 営業           | 名古屋   |
| 5  | 経理           | 東京     |

### projects（プロジェクト）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | 整数 | プロジェクトID（主キー） |
| name | 文字列 | プロジェクト名 |
| start_date | 日付 | 開始日 |
| end_date | 日付 | 終了日（NULL = 進行中） |
| budget | 整数 | 予算（円） |

| id | name     | start_date | end_date   | budget   |
|----|----------|------------|------------|----------|
| 1  | 基盤整備 | 2023-01-01 | 2023-12-31 | 5000000  |
| 2  | 新規開拓 | 2023-06-01 | 2024-05-31 | 3000000  |
| 3  | データ統合 | 2024-01-01 | NULL     | 8000000  |
| 4  | 海外展開 | 2022-01-01 | 2022-12-31 | 2000000  |

### employee_projects（社員とプロジェクトの対応）

社員とプロジェクトは「多対多」の関係です（1人の社員が複数プロジェクトに参加でき、1つのプロジェクトに複数社員が参加できる）。この関係を表すために中間テーブルを使います。

| カラム | 型 | 説明 |
|--------|-----|------|
| employee_id | 整数 | 社員ID |
| project_id | 整数 | プロジェクトID |
| role | 文字列 | 役割（'リーダー' または 'メンバー'） |

## 8. クエリの実行方法

このコースでは次の2つの方法でクエリを実行できます。

```bash
# ファイルに書いたSQLを実行する
psql "$DATABASE_URL" -f exercises/chapter01/ex01.sql

# 1行だけ試したいとき
psql "$DATABASE_URL" -c "SELECT * FROM employees;"

# 対話モード（psql シェルを起動）
psql "$DATABASE_URL"
```

VSCode を使っている場合は、サイドバーの「SQLTools」アイコンから `tokkun-sql` 接続を選び、SQL ファイルを開いて実行ボタンをクリックする方法もあります。
